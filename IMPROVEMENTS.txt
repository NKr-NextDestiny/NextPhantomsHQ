 Next Phantoms HQ — Verbesserungsvorschläge
 Stand: 2026-04-08
========================================


--- SICHERHEIT (HOCH) ---

1. ✓ .env ist bereits in .gitignore — korrekt umgesetzt

2. CSRF-Schutz fehlt
   - JWT sitzt in httpOnly Cookies, aber es gibt keinen CSRF-Token
   - Ein Angreifer könnte per Cross-Site-Request Aktionen im Namen eines Users ausführen
   - Fix: csurf Middleware oder Double-Submit-Cookie-Pattern
   - Aufwand: 2 Std

3. Rate-Limiting nur auf Auth-Routes
   - POST/PUT/DELETE Endpoints haben kein Rate-Limit
   - Jemand könnte z.B. hunderte Polls oder Trainings per Script erstellen
   - Fix: express-rate-limit auf alle Mutation-Endpoints
   - Aufwand: 30 Min

4. File-Upload-Validierung minimal
   - Kein Mime-Type-Check, nur Extension wird geprüft (.zip, .dem, .rec)
   - Malicious Files könnten durchkommen
   - Fix: Magic-Bytes/Mime-Type prüfen, nicht nur Dateiendung
   - Aufwand: 1 Std


--- PERFORMANCE ---

5. N+1 Queries in Training/Scrim-Erstellung
   - Dateien: server/src/routes/training.routes.ts, scrim.routes.ts
   - E-Mails werden in einer for-Schleife einzeln gesendet statt mit Promise.all
   - Fix: Promise.all(members.map(m => sendNotification(m)))
   - Aufwand: 30 Min

6. N+1 Queries im Scheduler
   - Datei: server/src/services/scheduler.service.ts
   - Trainings und Scrims werden einzeln in einer Schleife geladen
   - Fix: IDs sammeln, mit `where: { id: { in: ids } }` batch-laden
   - Aufwand: 30 Min

7. Fehlende Datenbank-Indexes
   - Datei: server/prisma/schema.prisma
   - Fehlende @@index auf: createdById (alle Models), userId (Comment),
     teamId+createdAt (für Zeit-Queries), pollId (PollOption)
   - Fix: @@index([createdById]) etc. hinzufügen + Migration
   - Aufwand: 30 Min

8. Keine Pagination auf Listen
   - Replays, Polls, Strats, Announcements — alles kommt ohne Limit
   - Bei vielen Einträgen wird das langsam
   - Fix: Cursor-basierte Pagination (skip/take oder cursor) + Client-seitig laden
   - Aufwand: 3 Std (alle Listen)

9. Kein Caching
   - Team-Config wird pro Request aus der DB geladen (requireFeature Middleware)
   - Fix: In-Memory-Cache mit TTL (z.B. 60s) für Team-Config und Game-Config
   - Aufwand: 1 Std


--- DEPLOYMENT / STABILITÄT ---

10. Kein Graceful Shutdown
    - Datei: server/src/index.ts
    - Server fängt SIGTERM/SIGINT nicht ab
    - Prisma-Verbindung wird nicht sauber geschlossen, laufende Requests brechen ab
    - Fix: process.on('SIGTERM', ...) mit prisma.$disconnect() und server.close()
    - Aufwand: 15 Min

11. Health-Checks fehlen in docker-compose.yml
    - Nur Postgres hat einen Health-Check
    - Server und Client haben keinen — Docker merkt nicht wenn sie hängen
    - Fix: healthcheck mit curl auf /api/health bzw. localhost:3000
    - Aufwand: 10 Min

12. Kein strukturiertes Logging
    - Überall nur console.log / console.error
    - Kein JSON-Logging, keine Log-Level, keine Request-IDs
    - Fix: pino oder winston Logger einführen
    - Aufwand: 2 Std

13. Socket.io-Emit Fehler werden verschluckt
    - 20+ Stellen mit `try { getIO().emit(...) } catch {}`
    - Wenn Socket.io nicht initialisiert ist, wird der Fehler komplett ignoriert
    - Fix: Mindestens console.error loggen
    - Aufwand: 15 Min


--- FEHLENDE FEATURES ---

14. Socket.io Client-Integration
    - Server broadcastet Events (training:created, match:updated, poll:vote, etc.)
    - Aber KEIN Client hört zu — socket.io-client ist installiert aber ungenutzt
    - Fix: useSocket Hook erstellen, in Pages subscriben, Daten live aktualisieren
    - Aufwand: 3 Std

15. Playbooks UI fehlt
    - Server hat komplette CRUD-Endpoints unter /api/strats/playbooks/*
    - Aber es gibt keine Client-Seite dafür
    - Fix: Playbooks-Page mit Liste, Erstellen/Bearbeiten, Strats zuordnen
    - Aufwand: 3 Std

16. E-Mail-Benachrichtigungen lückenhaft
    - E-Mails nur bei Event-Erstellung (Training, Scrim)
    - Fehlt für: Event-Änderungen, Löschungen, Poll-Ergebnisse, neue Announcements
    - Fix: Notification-Service erweitern
    - Aufwand: 2 Std


--- CODE-QUALITÄT ---

17. Error Boundary im Client fehlt
    - Datei: client/src/app/(app)/layout.tsx
    - Async-Fehler in useEffect können die ganze Seite crashen
    - Fix: React Error Boundary Komponente um das Layout wrappen
    - Aufwand: 30 Min

18. Zod-Validierung inkonsistent
    - Manche Routes nutzen validate() Middleware (training, strat, poll)
    - Andere parsen req.body direkt ohne Validierung (admin, comments teilweise)
    - Fix: Für alle POST/PUT Endpoints Zod-Schemas + validate() Middleware
    - Aufwand: 2 Std

19. Keine Tests
    - vitest ist in beiden package.json installiert, aber 0 Testfiles existieren
    - Fix: Mindestens Auth-Flow, Rollen-System und Kern-Business-Logik testen
    - Aufwand: 1-2 Tage
